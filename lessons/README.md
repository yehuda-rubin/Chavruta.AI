# Lessons — מחולל שיעורים מבוסס-ראג

מערכת לבניית שיעורי בית-מדרש מלאים מתוך הראג של Chavruta.AI. כל שיעור מורכב משני קבצים,
בדיוק כמו בישיבה: **דף מקורות** ו**מהלך שיעור**.

## הכלל המרכזי 🚫
בשלב זה **המודל (Claude) הוא המלמד** — הוא מסדר את המקורות, בונה את הנרטיב, השאלות והמסקנות.
**אין שימוש ב-API של מודל חיצוני (Nebius) בכלל.** הראג משמש לאחזור מקורות בלבד
(`scripts/retrieve_sources.py`), והמודל כותב מהם. המטרה: להוכיח שאחזור איכותי + מודל = שיעור
איכותי, לפני שמוציאים תקציב API.

## מבנה
```
lessons/
├── templates/
│   ├── TEMPLATE_source_sheet.md   ← תבנית דף מקורות (7 שערים: מקרא→משנה→גמרא→ראשונים→נו"כ→שו"ע→פוסקים)
│   └── TEMPLATE_lesson_flow.md    ← תבנית מהלך שיעור (פתיחה→יסוד→סוגיה→ראשונים→ליבון→הלכה→סיכום)
└── kriat-shema-arvit/             ← שיעור לדוגמה מלא (מאומת מול הראג)
    ├── source_sheet.md            ← דף מקורות: 11 מקורות, 7 שכבות
    └── lesson_flow.md             ← מהלך שיעור: 60 דק', 6 שלבים
```

## איך בונים שיעור חדש
1. **אחזור מקורות** — הרץ שאילתות ממוקדות מול הראג:
   ```bash
   .venv/Scripts/python.exe scripts/retrieve_sources.py --k 5 --full \
       --query "<לב הסוגיה>" --query "<שיטת ראשון>" --query "<פסק שו"ע>"
   ```
   (אחזור בלבד — אין קריאת LLM. הפלט: ref + טקסט מלא + deep_link לכל מקור.)
2. **העתק תבנית** מ-`templates/` לתיקיית שיעור חדשה.
3. **מלא את דף המקורות** מפלטי הראג — שמור על הסדר ההיררכי המסורתי; כל מקור עם ref וקישור אמיתי.
4. **כתוב את מהלך השיעור** — המודל בונה פתיחה מניעה, שאלות עיון, ליבון השיטות, מסקנה ונפק"מ,
   תוך הפניה למספרי המקורות `[N]`.
5. **אמת** — כל ציטוט מול ה-deep_link שבדף המקורות.

## המבנה הפדגוגי (מבוסס על אתרי הישיבות)
נלמד ממבנה השיעורים של הר עציון (תחומים היררכיים: תנ"ך/תלמוד/מחשבה/הלכה, שיעורים כתובים +
דפי מקורות), ישיבת בית-אל (ישיבה.org.il), כרם ביבנה, מעלה אדומים ורמת גן. ה-DNA המשותף:
שיעור נבנה סביב **שאלה**, מתקדם דרך שכבות המקורות (מקרא→משנה→גמרא→ראשונים→אחרונים→פסק),
ומסתיים בנפקא-מינה למעשה.
