# Lessons — מערכת התבניות (Template RAG) של Chavruta.AI

מנוע בניית שיעורי בית-מדרש מבוסס **שתי שכבות אחזור**:

1. **איזו תבנית?** — RAG של תבניות (`chavruta_templates`): מאחזר את התבנית המתאימה ביותר לנושא
   השיעור (`scripts/retrieve_template.py`).
2. **אילו מקורות?** — RAG של התוכן (`chavruta`, ~2.75M מקורות): מאחזר את המקורות עצמם
   (`scripts/retrieve_sources.py`).

המערכת מרכיבה מהם שיעור. **המודל (Claude) הוא המלמד** — הוא ממלא את התבנית מן המקורות.
🚫 **אין שימוש ב-API של מודל חיצוני (Nebius) בכל התהליך.**

## מה זו "תבנית"
כל תבנית = **תיקייה אחת עם 3 קבצים + manifest**:
```
lessons/templates/<genre>/
├── manifest.yaml            ← מטא-דאטה לאחזור (title, description, when_to_use, keywords) — זה מה שמוטמע ב-RAG
├── TEMPLATE_source_sheet.md ← דף מקורות (שערים לפי הז'אנר)
├── TEMPLATE_lesson_flow.md  ← מהלך השיעור (שלבים, שאלות, תזמון)
└── TEMPLATE_full_lesson.md  ← השיעור המלא (פרוזה מההתחלה עד הסוף)
```

הניתוב בין הז'אנרים נבחר אוטומטית ע"י `retrieve_template.py` לפי נושא השיעור (מאומת: 10/10 ז'אנרים
מנותבים נכון לשאילתות טיפוסיות).

## שני מצבים: שיעור ושו"ת (`mode`)
לכל תבנית שדה `mode` ב-manifest:
- **`mode: lesson`** (ברירת מחדל) — 3 קבצים (source_sheet, lesson_flow, full_lesson). 10 הז'אנרים למעלה.
- **`mode: shut`** — **פלט יחיד** (`TEMPLATE_answer.md` = מלל התשובה בלבד). למצב השו"ת של האפליקציה.

מסננים לפי מצב: `retrieve_template.py --mode shut` (רק תבניות שו"ת) / `--mode lesson` / `--mode all`.

### תבניות שו"ת (3)
| תבנית | מתי | מהלך התשובה |
|-------|-----|--------------|
| `shut-pesak/` | הוראה למעשה | מקור והגדרה → שיטות ראשונים → פוסקים → הכרעה למעשה |
| `shut-tzdadim/` | צד היתר וצד איסור | הגדרת הנדון → צד האיסור → צד ההיתר → הכרעה |
| `shut-birur-din/` | בירור והגדרת דין | מקור והגדרה → שיטות ראשונים → יישום פוסקים → מסקנה |

(ניתוב שו"ת מאומת 3/3. תשובות בדיקה לדוגמה: `sample_lessons/shut_*` — מקומי, לא בגיטהאב.)

## הצינור לבניית שיעור חדש
```bash
docker compose up -d qdrant

# פעם אחת (או אחרי שינוי תבניות): בנה/רענן את RAG התבניות
.venv/Scripts/python.exe scripts/index_templates.py

# 1) איזו תבנית מתאימה לנושא?
.venv/Scripts/python.exe scripts/retrieve_template.py --query "<נושא השיעור>"

# 2) שלוף מקורות לנושא (retrieval בלבד — אין LLM)
.venv/Scripts/python.exe scripts/retrieve_sources.py --k 5 --full --query "<לב הסוגיה>" --query "..."

# 3) המודל ממלא את 3 קבצי התבנית מן המקורות (Claude, ללא API).
# 4) מאמתים כל ציטוט מול ה-deep_link.
```

## שיעורים (בדיקה בלבד)
שיעורים שנבנים לבדיקת התבניות נשמרים ב-`sample_lessons/` בשורש הפרויקט, שהיא **תיקייה
מקומית ב-.gitignore** — היא לעולם אינה נדחפת לגיטהאב. הרפוזיטורי מכיל את **התבניות והכלים**, לא
את השיעורים. (התבניות אומתו ע"י שיעורי בדיקה: general → קריאת שמע/מוקצה/חנוכה; parasha →
בריאת האור; machshava → בחירה חופשית; mussar → כעס.)

## הרחבה
כדי להוסיף ז'אנר: צור `lessons/templates/<genre>/` עם 3 קבצים + `manifest.yaml`, הרץ
`index_templates.py`, ואמת עם שיעור בדיקה אחד ב-`sample_lessons/`.
