# תוכנית עבודה — הבנת שאלות עקיפות + הבאת חומר קשור

**יעד:** שהמערכת תבין שאלות עקיפות/אנליטיות (כמו *"מה המחלוקת בין רש"י לרמב"ן
בפסוק הראשון בתורה?"*) ותביא בוודאות את המקורות הנכונים + חומר קשור — במקום לענות
"אין במקורות".

מסמך זה מתעדכן תוך כדי. רכיבים ב-`src/chavruta/`.

> **סטטוס:** Phase 1 + 2 + 3 **בוצעו, נבדקו, ואומתו end-to-end מול ה-backend החי**
> (59/59 טסטים עוברים). השאלה *"מה המחלוקת בין רש"י לרמב"ן בפסוק הראשון בתורה?"*
> מחזירה כעת תשובה **מעוגנת המצטטת את שני המקורות**: `Rashi on Genesis.1.1` +
> `Ramban on Genesis.1.1`. ✅ הגדרת ה"בוצע" הושגה.
>
> שלושה תיקונים בדרך: (1) Phase 1 — זיהוי הפניות עברית; (2) Phase 2 — ציוני דרך;
> (3) Phase 3 — אחזור מודע-עוגן: פנייה ממוקדת למפרשים הנקובים על הפסוק + **תיקון
> `_dedup` לשמור את הציון הגבוה** (אחרת מפרש עוגן מודח ע"י כפיל vector בציון נמוך).
> **Phase 4 בוצע:** גרף קישורים נבנה ([scripts/build_links.py](../../scripts/build_links.py)
> → `data/links.jsonl`: 340,603 קשתות, 500,855 צמתים) וה-`LinkExpander` פעיל —
> COMPARE/LESSON מרחיבים מהפסוקים-עוגן לחומר קשור (מפרשים + סופר-פירושים).
> **בונוס:** תוקן `enforce_citations` לתמוך בסמנים משולבים (`[S1, S2]`) — עיגון אמין
> ללא תלות בפורמט שה-LLM בוחר. 68/68 טסטים עוברים.
>
> **Phase 5 בוצע:** מתכנן-שאילתה מבוסס-LLM ([llm_planner.py](../../src/chavruta/intents/llm_planner.py))
> מאחורי הדגל `CHAVRUTA_QUERY_PLANNER=llm` — fallback שרץ רק כשההיוריסטיקה לא מצאה
> הפניה, ולעולם לא שובר בקשה (כשלים נבלעים). ברירת-מחדל off (אופליין/דטרמיניסטי).
>
> **Phase 6 בוצע:** סט-הערכה ([tests/eval/indirect_questions.jsonl](../../tests/eval/indirect_questions.jsonl),
> 15 שאלות עקיפות/אנליטיות) + שער-רגרסיה ([eval_recall.py](../../scripts/eval_recall.py)
> ו-[test_recall_gate.py](../../tests/eval/test_recall_gate.py)) — **recall@router = 15/15 (100%)**,
> שער 90%. ה-reranker (`CHAVRUTA_RERANK=true`) זמין כדגל לחידוד דיוק (cross-encoder כבד,
> כבוי כברירת-מחדל; הסט-הערכה מודד את השפעתו כשמופעל).
>
> **כל 6 השלבים הושלמו. 86/86 טסטים עוברים.**

## ניתוח שורש (מבוסס קוד קיים)

| ממצא | קובץ | משמעות |
|------|------|--------|
| `detect_refs` מזהה רק שמות ספרים **באנגלית** (`Genesis\|Exodus…`) | [router.py](../../src/chavruta/intents/router.py) | שאלה בעברית ("בראשית א:א", ובוודאי "הפסוק הראשון") **לעולם** לא הופכת ל-`named_refs` |
| בלי `named_refs` → אין anchoring | [hybrid.py](../../src/chavruta/retrieval/hybrid.py) | האחזור מנסה להתאים סמנטית את הניסוח האנליטי → מפספס את הפירושים בפועל |
| `commentator_ids` → **פילטר קשיח** | hybrid `_filters` | מצמצם לרש"י/רמב"ן אך בלי עוגן-פסוק → מביא פירושים לא-נכונים שלהם |
| ה-payload כולל **`anchor_ref`**, ו-`fetch_by_refs` תופס לפי ref **או** anchor_ref | [qdrant_store.py](../../src/chavruta/store/qdrant_store.py) | ✅ ברגע ש-`named_refs=["Genesis.1.1"]` — המערכת כבר תמשוך את הפסוק **+ כל המפרשים עליו** בציון 1.0 → מעוגן |
| `data/links.jsonl` **לא קיים** → `link_expander=None` | [pipeline.py](../../src/chavruta/pipeline/pipeline.py) | הרחבת "חומר קשור" (שרשרת מסירה / סופר-פירושים) **כבויה לגמרי** |

**מסקנה:** מנוע ה-anchoring כבר עובד. החסר העיקרי הוא **הבנת-שאילתה** שתפיק `named_refs`
מעברית ומרמזים עקיפים. זהו ה-leverage הגבוה ביותר ונותן את רוב בקשת המשתמש.

## שלבים

### Phase 1 — זיהוי הפניות בעברית + מובנות  ⟵ הזוכה הגדול, סיכון נמוך
הרחבת `detect_refs` לזהות:
- **שמות ספרי תנ"ך בעברית** (בראשית→Genesis, שמות→Exodus, … 39 ספרים).
- **שמות מסכתות** (בבא מציעא→Bava_Metzia, ברכות→Berakhot …) + פורמט דף (ב׳→2a, ג׳→3a).
- **פרק/פסוק בעברית**: "בראשית א׳:א׳", "בראשית פרק א פסוק א", גרשיים/גימטריה (א=1 … ט"ו=15) וגם ספרות.
- מיפוי לפורמט הקורפוס `Book.ch.verse` / `Tractate.daf`.

**קבצים:** `router.py` (טבלאות מיפוי he→en + רגקס עברי), `intents/hebrew_refs.py` (חדש, נתונים).
**טסט:** טבלת קלט→פלט (≥40 דוגמאות), כולל גרשיים, טווחים, מסכתות.

### Phase 2 — מפענח "ציוני דרך" / הפניות עקיפות  ⟵ ליבת "שאלות עקיפות"
מפה נתונית (JSON/YAML שהמשתמש יכול להרחיב) של ביטויים מוכרים → הפניה:
- מוחלט: "הפסוק הראשון בתורה"→`Genesis.1.1`, "עשרת הדיברות"→`Exodus.20`, "קריאת שמע"→`Deuteronomy.6.4`, "פרשת בראשית"→`Genesis.1`.
- יחסי: "הפסוק/המשנה/הדף ה{ראשון|אחרון} ב{work}", "תחילת/סוף {work}".
מזין `named_refs`. אם זוהה ביטוי יחסי — רזולוציה מול הרישום/קורפוס.

**קבצים:** `intents/landmarks.py` + `data/landmarks.yaml` (חדש), חיבור ב-`router.route`.
**טסט:** הביטויים המוכרים → ref נכון; שאילתת רש"י/רמב"ן הופכת ל-`named_refs=["Genesis.1.1"]`.

### Phase 3 — מדיניות אחזור מודעת-עוגן  ⟵ שגם שני המפרשים יופיעו
- כש-`named_refs` קיים **וגם** שמות מפרשים: לא לפלטר קשיח את חיפוש הווקטור לפי מפרש
  (מרעיב recall); במקום זאת למשוך את הפסוק + המפרשים העוגנים דרך `fetch_by_refs`,
  ולהטות/לבוסט את המפרשים הנקובים.
- לוודא ששני המפרשים (רש"י+רמב"ן) על אותו פסוק נוכחים; בדיקת ה-compare-honesty הקיימת
  כבר תדווח אם אחד חסר.

**קבצים:** `hybrid.py` (commentator_ids כ-bias כשיש עוגן), `router.py` (לא לפלטר ב-COMPARE עם עוגן).
**טסט:** "מחלוקת רש"י/רמב"ן בבראשית א:א" → hits כוללים את שניהם על הפסוק.

### Phase 4 — גרף קישורים לחומר קשור (LinkGraph)  ⟵ "להביא דברים קשורים"
- סקריפט `scripts/build_links.py` שמייצר `data/links.jsonl`: קשת
  `anchor_ref → ref` לכל פירוש (כבר ניתן מה-payload), ובמידת האפשר שרשרת-מסירה
  (פסוק → ראשונים → אחרונים → הלכה) ממיפויי Sefaria.
- ה-pipeline כבר טוען `links.jsonl` ומפעיל `LinkExpander` → COMPARE/LESSON ימשכו
  סופר-פירושים וסוגיות קשורות (depth 2 כבר מוגדר ב-router).

**קבצים:** `scripts/build_links.py` (חדש), פלט `data/links.jsonl`.
**טסט:** expand מ-`Genesis.1.1` מחזיר מפרשים + קישורים; `link_expander` לא None.

### Phase 5 — (אופציונלי) מתכנן-שאילתה מבוסס-LLM  ⟵ תקרת חוסן
לשאלות שההיוריסטיקה מפספסת: קריאת-Nebius קלה אחת שמחלצת JSON
`{refs, commentators, intent, sub_questions}` ומזינה `named_refs`/`commentator_ids`.
מאחורי דגל (`CHAVRUTA_QUERY_PLANNER=llm`); ברירת-המחדל נשארת היוריסטית/אופליין/דטרמיניסטית
(עקרון II). עלות/לטנציה נוספת.

**קבצים:** `intents/llm_planner.py` (חדש), חיבור מותנה-דגל ב-`router`/`pipeline`.

### Phase 6 — Rerank + מדידה
- להדליק את ה-cross-encoder reranker (`CHAVRUTA_RERANK=true`) לדיוק על מאגר-המועמדים המורחב.
- סט-הערכה קטן (10–20 שאלות עקיפות/אנליטיות עם הפניות צפויות) → למדוד recall@k לפני/אחרי
  כל שלב; שער-רגרסיה (עקרון איכות בחוקה).

**קבצים:** `tests/eval/indirect_questions.jsonl` (חדש), `scripts/eval_recall.py` (חדש).

## סדר ביצוע ומאמץ
1. **Phase 1 + 2** — נותן את רוב הבקשה (שאלה עקיפה → פסוק נכון → רש"י/רמב"ן עוגנים). מאמץ בינוני.
2. **Phase 3** — מבטיח ששני המפרשים עולים. מאמץ קטן.
3. **Phase 4** — מרחיב ל"חומר קשור" אמיתי. מאמץ בינוני-גבוה (תלוי בנתוני קישור).
4. **Phase 5/6** — ליטוש וחוסן. אופציונלי.

## סיכונים
- פענוח הפניות עברית עדין (גימטריה, טווחים, דפי גמרא).
- מפת ציוני-הדרך עתירת-תחזוקה (אבל נתונית וניתנת להרחבה).
- בניית גרף הקישורים תלויה בזמינות נתוני הקישור של Sefaria.

## הגדרת "בוצע"
השאלה *"מה המחלוקת בין רש"י לרמב"ן בפסוק הראשון בתורה?"* מחזירה תשובה **מעוגנת**
המצטטת את רש"י **ו**רמב"ן על `Genesis.1.1`, ו-≥80% מסט-ההערכה מחזיר את ההפניה הצפויה.
