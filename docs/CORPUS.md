# Chavruta.AI — Corpus Scope

היקף הטקסטים שהמערכת מאחזרת מהם. מקור: **Sefaria** (API + bulk export). כל טקסט נשמר
**בעברית ובאנגלית** (דרישה #5).

### + שלושה (ומעלה) שבחרתי — והנימוק
> **הנימוק המרכזי:** רוב המפרשים שביקשת הם **תורה בלבד**. כדי לכסות באמת את *כל* התנ"ך
> צריך מפרשים של **נביאים וכתובים** — אחרת נ"ך יישאר חשוף. לכן בחרתי כך שייסגר הפער:

| מפרש | כיסוי | למה נבחר |
|------|-------|-----------|
| **רשב"ם** | תורה | פשט קלאסי, משלים את רש"י |
| **אור החיים** | תורה | עומק ודרש — קול "רב" עשיר |
| **רד"ק** | **נביאים + כתובים** (+תורה) | ⭐ חיוני — המפרש המרכזי לנ"ך |

| **מצודת דוד / ציון** | **נביאים + כתובים** | הפירוש הסטנדרטי ללימוד נ"ך |

כך מתקבל כיסוי מלא: רש"י/רשב"ם/אבן-עזרא/רמב"ן/ספורנו/בעה"ט/אוה"ח לתורה, ורד"ק/מצודות/מלבי"ם/רש"י לנ"ך.

## 3. זמינות ב-Sefaria ושליפה
- **Texts API** — `GET /api/v3/texts/{ref}` → טקסט עברי+אנגלי לכל הפניה.
- **Links API / Linker** — קישורים והפניות בין מקורות (לשכבת האחזור).
- **Bulk export** — מאגר Sefaria-Export (GitHub) להורדה חד-פעמית → תומך בפרופיל ה-offline (#8).
- כיסוי משתנה: לא לכל פסוק יש כל מפרש (למשל ספורנו/אוה"ח = תורה בלבד). זה תקין — מאחזרים מה שקיים.

## 4. הערכת היקף (Scale)
| | נוכחי (תורה) | יעד (כל התנ"ך + ~12 מקורות) |
|---|---|---|
| פסוקים | ~5,846 | ~23,000 |
| chunks (משוער) | 15,274 | ~150K–300K |

הטמעת קורפוס בסדר גודל כזה = **Nebius embedding job** (חד-פעמי, pay-per-use). מקומית ניתן
להטמיע בהדרגה (ספר-ספר) על ה-CPU.
